Git quick reference

covers everyday branch, commit, history, and recovery commands — porcelain, not plumbing

Stage & commit

  • git add -pStage hunks interactively, not whole files
  • git commit --amend --no-editFold staged changes into the last commit, keep its message
  • git commit --fixup <sha>Mark a fixup for autosquash on the next rebase
  • git restore --staged <file>Unstage without touching the working tree

Branch & switch

  • git switch -c <name>Create a branch and move onto it
  • git switch -Jump back to the previous branch
  • git branch --mergedList branches already folded into HEAD — safe to delete
  • git push -u origin HEADPush the current branch and set its upstream

switch is the modern split of checkout; use restore for files.

Inspect history

  • git log --oneline --graphCompact topology of the branch graph
  • git log -p <file>Every change to one file, newest first
  • git log -S'<text>'Find commits that added or removed a string (pickaxe)
  • git blame -L 20,40 <file>Who last touched each line in a range

Diff & compare

  • git diffWorking tree vs. index — what you haven't staged
  • git diff --stagedIndex vs. HEAD — what the next commit holds
  • git diff main...Changes on this branch since it left main
  • git range-diff main HEAD@{1} HEADCompare two versions of a rebased branch

Undo & recover

  • git restore <file>Discard working-tree edits to a file
  • git reset --soft HEAD~1Undo the last commit, keep changes staged
  • git revert <sha>New commit that inverts an old one — safe on shared history
  • git reflogThe escape hatch: every HEAD move, even "lost" commits

reset --hard throws away work — confirm with git status first.

Rebase & rewrite

  • git rebase -i mainReorder, squash, and reword commits since main
  • git rebase --autosquashApply --fixup marks automatically
  • git rebase --abortBail out and restore the pre-rebase state
  • git push --force-with-leaseOverwrite a remote branch only if nobody else pushed

Never rebase a branch others have already pulled from.

Stash & worktree

stash push -m
Shelve dirty changes with a label
stash pop
Reapply the top stash and drop it
worktree add
Check a branch out in a second directory
worktree prune
Clean up stale worktree links

Revision syntax

HEAD~3
Third ancestor along the first parent
HEAD^2
Second parent of a merge
@{u}
The upstream of the current branch
main@{1}
Where main pointed one move ago

Interactive keys

  • yStage this hunk (in add -p)
  • sSplit the hunk into smaller ones
  • j/kNext / previous hunk
  • qQuit the interactive session
